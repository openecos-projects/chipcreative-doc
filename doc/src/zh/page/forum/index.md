# 创芯论坛
## 简介
创芯论坛（后文简称本论坛）是由ECOS团队创建并负责运营的一个半学术会议/活动，**旨在分享中国在开源芯片技术和生态发展上的最新动态。**与传统学术会议（如ASPLOS/HPCA/ISCA/MICRO计算机体系结构四大顶会）或技术峰会（如RISC-V峰会）不同的是，本论坛具有两个非常鲜明的特征：**一参会门槛与成本非常低。**无需计算机相关领域的学术背景或实践经验，任何对开源芯片感兴趣的爱好者均可免费投稿报告或参与线下交流讨论；**二坚持最纯粹的技术交流。**参与者均为一线科研工作者、产业界工程师、学生或完全不懂芯片技术的爱好者（ECOS团队认为此类用户十分可贵，他们往往能贡献最真实的需求和最富创造力的想法），讨论内容直击开源芯片技术发展与落地的核心关键问题，并彻底摒弃各种前摇冗长且毫无营养的“形式主义”。通过这种以社区为导向的组织形式，本论坛致力于营造一种纯粹、轻松的氛围，让每位参会者都**“敢于分享、乐于分享”**，以共同推动开源芯片向着正确的道路不断前进。

## 背景
ECOS团队建立本论坛的核心目的是**搭建一个对外展示中国开源芯片发展成果的平台，并联合国内外相关团队共同促进开源芯片生态发展。**《孙子兵法》中曾经提到过：**知己知彼，百胜不殆。**ECOS团队认为想要将开源芯片变成真正意义可以影响甚至引领未来科技发展的中坚力量，就必须要先做到“知己”，即系统性地分析和总结自身在该领域内所获得的成果、经验、理念等内容，随后再通过技术交流或知识共享的方式，让国内外具有相同研究兴趣的团队深入了解彼此，即做到“知彼”，从而为后续的产学研合作奠定坚实的基础。**能够成为激励个体进步和实现广泛交流合作的助推器，这便是创芯论坛存在的最大意义和价值所在。**希望未来的某一天，创芯论坛（ChipCreative Forum）可以跟欧洲的FSiC和ORConf，北美的Latch-Up，大洋洲的Down Underflow等会议并驾齐驱，向全世界讲好中国、亚洲、世界开源芯片的故事。

## 目标
本论坛聚焦跟开源芯片技术或生态有关的一切领域，如开源IP/SoC、开源EDA、开源PDK、开源硬件（PCB设计与嵌入式应用）、开源流片、芯片设计平台、先进封装、新型芯片、人才培养、社区建设等。从含金量和影响力两个维度出发，ECOS团队对本论坛未来发展目标给出了初步的设想和路线，具体内容如下：

- **聚焦中国（三年）：**主要面向国内开源芯片社区和爱好者。通过组织高质量的技术论坛和一系列同期活动，探索中国开源芯片技术与生态的最佳构建方式，满足国内科研机构、高校、社区在理论和技术探索等方面的交流合作需求，并优先支持开发者在本土环境下完成开源芯片的设计、验证和流片。该阶段重点聚焦于国内，因此本论坛形式不追求规模宏大、流程繁琐等外在表现，而专注于打通开源芯片全技术链路。
- **连结亚洲（三年）：**主要面向亚洲开源芯片社区和爱好者。在“聚焦中国”阶段所积累的经验上，吸引或邀请亚洲其他国家或区域的开源芯片团队参与本论坛，提升亚洲在开源芯片领域上的协同与创新能力，并将部分精力转移到构建区域性开源标准与合作层面，包括平台建设、商业落地、人才培养等，让亚洲开发者能够用较低的成本完成芯片的全流程设计。
- **走向世界（五年）：**主要面向全球开源芯片社区和爱好者。在“连结亚洲”阶段所收集的成功案例与国际协作模式上，进一步吸引或邀请全世界开源芯片领域的专家、代表、爱好者参与本论坛，并围绕不同国家或地区在开源芯片领域内的先发优势和所取得的技术成果，推广一系列可实施、可复现的典型解决方案，从而逐步促成开源芯片领域“一带一路”合作机制的最终建立，让全世界范围内的用户都能以更低的门槛切身体会到开源技术所带来的红利。

## 条款
本论坛倡导**开放、包容、务实**的交流环境。参会者在论坛期间及社区频道内进行技术分享与讨论时，应遵守基础的行为准则，**严禁发布、传播任何违反法律法规、误导性或虚假的错误言论，以及任何形式的歧视与人身攻击。**ECOS团队保留对违反社区准则的行为进行干预（包括但不限于取消参会资格、清理违规内容、加入失信黑名单等）的权利，以维护开源技术交流的纯粹性与专业性，确保每位成员都能在尊重与互助的氛围中获得成长。与此同时，每位参会者应自觉维护本论坛的品牌声誉，不得采用任何手段进行抹黑或污化，违者同样将被处罚。

关于知识产权与法律风险，**创芯论坛的品牌、组织策划及相关流媒体影像版权归ECOS团队所有，而报告讲义、技术文档及演讲内容的版权则归各演讲嘉宾个人或其所属单位机构所有。**未经ECOS团队或原创作者的明确书面授权，任何机构或个人均不得擅自将本论坛内容用于商业用途，或进行二次剪辑与镜像转载，如有发现，ECOS团队将保留法律追责的权力。此外，本论坛所分享的技术案例与解决方案仅供交流参考，因直接或间接引用相关技术产生的法律风险及损失，本论坛组织方不承担任何责任，请开发者在应用过程中自行遵循相关开源协议与合规要求。
